# AP 시스템 & 자원 관리

턴당 주어지는 **AP(Action Point)** 로 카드를 사용하는 전략적 자원 관리 시스템

### AP 코스트별 카드 분포

| AP | 카드 수 | 비율 | 구성 |
|----|---------|------|------|
| 0 | 15장 | 7.5% | 스킬 12, 강화 2, 공격 1 |
| 1 | 150장 | 75.0% | 공격 82, 스킬 54, 강화 14 |
| 2 | 34장 | 17.0% | 공격 32, 스킬 2 |

> AP 0 카드 중 80%가 스킬 — **무료 유틸리티** 설계 의도

# 데미지 공식

### ATK 기반 (일반 캐릭터)

```
최종 피해 = CardMultiplier × ATK × 0.35 × modifiers
```

**예시**: 레노아 (ATK 492) — 피해 140% 카드
`1.40 × 492 × 0.35 = 241.1`

### DEF 기반 (나인 등 Vanguard)

```
최종 피해 = CardMultiplier × ((ATK × 0.3) + (DEF × 2.1)) × 0.35
```

**예시**: 나인 (ATK 407, DEF 178) — 피해 100% 카드
`1.00 × ((407 × 0.3) + (178 × 2.1)) × 0.35 = 1.00 × (122.1 + 373.8) × 0.35 = 173.6`

### 크리티컬 기대값 보정

```
기대 배율 = 1 + (CC/100) × ((150 + CD)/100 - 1)
```

기본 CC=10%, CD=12% → `1 + 0.10 × (1.62 - 1) = 1.062` (+6.2%)

# 태그 공존 분석

같은 카드에 **2개 이상 태그**가 붙은 경우의 조합 빈도

| 태그 조합 | 카드 수 | 해당 카드 |

| **개전 + 유일** | 4장 | 파쇄, 황혼의 결속, 화룡의 보석, 발사 준비 |
| **소멸 + 증발** | 2장 | 파쇄, 납도 |

### 핵심 패턴

- **개전 + 유일** = 가장 빈번한 조합 — 턴 시작에 고유 효과를 발동하는 **핵심 강화 카드** 패턴

> **QA 포인트**: 파쇄 한 장에 태그 상호작용이 집중 — 태그 발동 순서/우선순위 **엣지 케이스** 다수

# 분석 요약 & QA 시사점

### 핵심 발견

1. **태그는 고유 카드 전용 시스템** — 200장 중 31장(15.5%)만 태그 보유, 전부 고유 카드
2. **유일 태그 = 캐릭터 아이덴티티** — 강화 타입에 집중, 각 캐릭터 고유 매커니즘 정의
3. **파쇄(나인) = 태그 시스템의 교차점** — 4중 태그, AP 3, 350% 배율의 올인 카드
4. **개전 + 유일 = 핵심 설계 패턴** — 턴 시작에 고유 강화 효과 발동

### QA 집중 테스트 영역

| 영역 | 테스트 포인트 |
|------|-------------|
| 태그 발동 순서 | 복수 태그 카드의 효과 실행 순서 (파쇄 4중 태그) |
| 보존 + 핸드 제한 | 보존 태그 카드가 핸드 꽉 찼을 때의 처리 |
| 증발 카드 재생성 | 증발 후 카드 복구 불가 조건 검증 |
| 다단 히트 × 크리티컬 | 히트별 개별 크리 판정의 DPS 편차 |
| 개전 동시 발동 | 복수 개전 카드 보유 시 발동 충돌 |
| 기본/고유 경계 | 태그가 기본 카드와 상호작용하는 경우 |
